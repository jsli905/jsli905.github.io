National Institute of Standards and Technology (NIST) - Additive Manufacturing Research Intern

Dates: June 2019 – August 2020
Team: Engineering Lab, Additive Manufacturing Project
Developed software tools, computer vision pipelines, and image analysis methods for additive manufacturing research at the National Institute of Standards and Technology.

Additive manufacturing offers transformative potential for defense, aerospace, and medical part production — but adoption is bottlenecked by the complete absence of public qualification standards. Without qualification, each organization must conduct its own costly, time-consuming testing programs, and data almost always remains proprietary.

  • No public qualification framework exists for AM processes or materials in aerospace/defense
  • Internal defects (gas pores, lack-of-fusion, cracking, warping) occur during L-PBF and can cause part failure if undetected
  • Layerwise images captured by the AMMT's in-situ camera came out rotated and perspective-distorted due to a 63.2° off-axis camera angle — unusable for accurate defect localization without correction
  • The AM Material Database (ammd.nist.gov) required manual XML curation of experimental data — a slow, error-prone process not scalable to the volume of tests needed for qualification
  • Existing test data from other labs remained siloed; a shared, structured public database was needed to pool results toward qualification

Additive manufacturing generates large amounts of process data that can be used to improve material characterization, process understanding, and defect detection. During my time at NIST, I developed software tools to automate data handling, process experimental data, and analyze manufacturing images collected during laser powder bed fusion builds.

Role Summary

During three appointments with NIST's Engineering Laboratory, I worked on projects supporting additive manufacturing data management, process monitoring, and image analysis. My work ranged from automating data curation workflows for the Additive Manufacturing Materials Database (AMMD) to developing computer vision algorithms for processing layerwise images collected during metal additive manufacturing builds. These projects combined software development, data engineering, image processing, and manufacturing research.

AM Material Database — XML Data Curation Automation

June-September 2019

The Additive Manufacturing Materials Database (AMMD) stores material and process data from additive manufacturing experiments. Researchers previously required researchers to manually convert Excel-based experimental results into structured XML before upload. I wrote Python scripts to automate this conversion pipeline: queried file repositories, automatically located matching datasets, downloading and unzipping them, parsing Excel data, and generating XML files that are uploaded to the database.

  • Automated file discovery via query, download, and unzip workflow
  • Converted Excel experimental data to structured XML at scale
  • Reduced manual effort for data curation and database population

Melt Pool Characterization from In-Situ Print Images

December 2019

Melt pool geometry is a key indicator of print quality in L-PBF adn directly influences part microstructure and mechanical properties. Improper melt pool formation underlies the majority of fusion defects and trapped-gas porosity. I developed image-processing Python programs to extract melt pool properties from high-speed camera images captured during Additive Manufacturing Metrology Testbed (AMMT) printing tests, feeding quantitative data into NIST's model-based qualification effort. The programs extracted geometric measurements and process metrics that could be used by researchers to analyze process stability and correlate printing parameters with manufacturing outcomes.

  • Wrote image processing programs to measure melt pool size and morphology from in-situ images
  • Supported the Engineering Laboratory's model-based AM qualification research program

Layerwise Image Processing via Computer Vision

Developed a full computer vision pipeline to correct distorted layerwise images from the AMMT's angled layer camera, enabling accurate real-world localization of defects. Benchmarked six undistortion methods against synthetic ground-truth images generated from XYPT build command data, then applied the best-performing approach to the second experiment's dataset, cropping and sorting images by part and layer for analysis.

  • Tested and compared six OpenCV-based image undistortion methods (getPerspectiveTransform, findHomography variants)
  • Built synthetic ground-truth images from XYPT build command files for quantitative benchmarking
  • Designed automated pipeline to undistort, crop, and sort layerwise images by part/layer
  • Identified recurring defect patterns: elevated edges, corner protrusion through powder, subsurface blobs

Requirements & Constraints

  • Accuracy: Undistorted images needed to match real-world coordinates accurately enough to localize defects for closed-loop feedback to the AMMT during printing
  • Scalability: XML curation automation had to handle large batches of experimental files without manual intervention per file
  • Image quality constraints: Low contrast between metal parts and powder background limited automated edge detection; corner-point selection required manual assistance
  • Instrument constraints: Layer imaging camera fixed at 63.2° from surface normal with viewing vector <0.45, 0, 0.89> — perspective distortion was a fundamental property of the setup, not an error
  • Data format constraints: Build command data stored in XYPT CSV files (x/y coordinates, laser power, trigger) at 10 µs resolution — synthetic image generation required parsing these correctly
  • Research scope: Work fed into a larger multi-lab qualification program; outputs needed to be documented and reproducible for other researchers

Results & Impact

The automation pipeline for XML curation eliminated manual data preparation steps, enabling faster and more reliable contributions to the publicly accessible AM Material Database. The layerwise image pipeline produced corrected, cropped, and organized image datasets ready for downstream machine learning and defect classification work. Defect pattern findings (elevated edges, corner protrusion, subsurface blobs) gave the AMMT team actionable hypotheses for parameter adjustment and future automated image analysis and in-situ defect corretion work.
